Retrieval Augmented Generation (RAG)

Click each step to understand exactly what happens inside a RAG application. This follows the same explanation you described—but with a little more technical accuracy.

📚

1. Build the Knowledge Base

First we collect documents such as PDFs, manuals, SharePoint files, Confluence pages or databases. These documents become our Knowledge Base.

3. Create Embeddings

Each chunk is converted into a numerical representation called an Embedding Vector. Similar meanings produce similar vectors.

"Reset Password"

[0.12,0.54,0.82...]
"Login Help"

[0.15,0.59,0.79...]
🗄️

4. Store Inside a Vector Database

The vector database stores:

  • The original text chunk
  • The embedding vector
  • Metadata (document, page, author, etc.)

5. User Asks a Question

Example: "How do I reset my Volvo account?"

How do I reset my account?
🔍

6. Convert Query into an Embedding

The user's question is converted into an embedding using the same embedding model. Now the query can be compared against all stored vectors.

Question ↓ Embedding ↓ [0.11,0.53,0.80...]
🎯

7. Similarity Search

The vector database compares the query embedding with every stored embedding. It returns the Top-K Most Similar Chunks.
